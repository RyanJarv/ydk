# ydk Example Directions

This directory contains fictional usage sessions that explore different directions for `ydk`.

- [Direct CLI](./direct-cli.md): `ydk` as a maintainer-facing terminal tool for answering why files, modules, and decisions exist.
- [CI Guardrails](./ci-guardrails.md): `ydk` as a pull request check that keeps purpose anchors and graph changes from drifting.
- [Coding Agent Context](./coding-agent-context.md): `ydk` as structured repo context that coding agents query before and after code changes.
- [Library Integration](./library-integration.md): `ydk` as a model format and API used by IDE extensions, PR bots, dashboards, or other developer tools.
